# 0009 — Explicit edges are cleaned up on re-ingestion, not just extracted ones

## Status
Accepted

## Context
§4.1's `SinkAdapter.delete_extracted_for_doc` is scoped, by name and docstring, to `origin='extracted'` only — silent on `origin='explicit'` (e.g. Obsidian wikilinks). If a wikilink is removed from a note's body and the note is re-ingested, the stale explicit edge would survive forever under the spec as originally written. This mirrors a real bug in NAA today: `upsert_relationships` only ever adds relationships via `MERGE`, never removes ones no longer present in the source text.

## Decision
The sink method's scope broadens to cover both `origin='extracted'` and `origin='explicit'` elements sourced from `doc_id` — both get removed before the doc's fresh extraction is rewritten. `origin='curated'` remains untouched, as already specified in §6.2. Rename the method's contract (not necessarily its name) to reflect "remove everything this doc previously contributed that isn't curated," rather than "remove only extracted."

## Consequences
- Wikilinks are ground truth *from the document* — if the document no longer says it, the graph shouldn't either. Removing this staleness is a deliberate improvement over NAA's current behavior, not a lift.
- Combined with ADR-0010 (delete tombstoning), a human who has manually deleted an explicit edge via the canvas won't have it silently regenerated by this cleanup-and-rewrite cycle.
